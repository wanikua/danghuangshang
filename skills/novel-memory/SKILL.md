# 小说记忆系统 — OpenViking 集成指南

你已被注入 `novel-memory` 技能。本技能指导如何使用 OpenViking 作为小说创作的三维记忆系统。

---

## OpenViking 三维映射

| OpenViking 模块 | 对应功能 | 存储内容 |
|----------------|---------|---------|
| **Memories** | 静态知识 + 动态日志 | 角色设定、世界观、章节摘要、角色状态、伏笔追踪 |
| **Resources** | 参考素材库 | 参考小说库、风格范本 |
| **Skills** | 结构化知识图谱 | 人物关系网络、世界设定体系 |

---

## 1. Memories — 核心记忆

### 存储内容

**静态设定（长期有效）：**
- 人物档案：姓名、年龄、外貌、性格、背景、动机
- 世界观规则：魔法体系、科技水平、政治制度、地理环境
- 组织信息：阵营、势力、层级结构

**动态日志（按时间推进）：**
- 章节摘要：每章核心事件（3-5 要点）
- 角色当前状态：位置、情绪、关系变化、持有物品
- 伏笔清单：已埋设但未回收的伏笔（标注埋设章节和预计回收章节）
- 时间线：故事内时间进度

### 使用时机

- **每章开始前（必读）**：读取上一章摘要、角色状态、未回收伏笔、时间线
- **每章完成后（必写）**：存储本章摘要、角色状态变化、新伏笔、时间线进度
- **新书启动时（初始化）**：创建初始设定记录

### 操作原则

- **初始化时记录**：定义新实体/规则时立即存入
- **更新时同步**：设定演变时及时更新
- **避免冗余**：只记录核心的、长期有效的信息
- **防遗忘**：防止第 100 章时忘记第 1 章的设定

---

## 2. Resources — 参考素材

### 使用场景

- 自主创作模式启动时（确定主题和风格）
- 卡文需要灵感时
- 参考特定场景描写方式时
- 学习叙事风格时

### 工作流

1. 根据需求描述检索相关素材
2. 分析检索结果：韵味、节奏、手法、切入点
3. 参考风格和手法，用自己的内容重新创作
4. **绝不照搬，只取灵感**

---

## 3. Skills — 知识图谱

### 存储内容

- **人物关系网络**：角色间的关系类型和变化历史
- **世界设定体系**：力量体系规则、社会结构、地理分布
- **因果链追踪**：事件间的因果关系图

### 使用时机

- 架构设计阶段：构建初始知识图谱
- 审核阶段：验证设定一致性
- 章节完成后：关系变化时更新图谱

---

## 工具组合策略

### 写作循环（标准流程）

```
开始新章 → Memories(读) + Skills(查) → 写作 → Memories(写) + Skills(更新)
```

### 自主创作启动

```
Resources(检索灵感) → Memories(建立设定) + Skills(构建图谱) → 开始写作
```

### 一致性验证

```
Memories(核对设定) + Skills(检查关系) → 发现冲突 → 修正
```

---

## 注意事项

1. **模块职责分明**：Memories 存数据，Resources 检索参考，Skills 管结构
2. **跨模块一致性**：更新 Memories 中的设定时，检查 Skills 中的关系是否需要同步
3. **定期维护**：清理过时信息，保持记忆系统有效性
4. **OpenViking 安装/配置**：不在本 skill 范围内，请参考 OpenViking 官方文档
