## 一、项目概述

### 核心理念
我开发了一套名为"AI朝廷"的多Agent协作系统，灵感来自中国明朝的六部制度，将不同专业领域的AI模型组织成一个协同工作的团队。这不仅是一个技术项目，更是一次将传统文化智慧与现代AI技术结合的创新实践。

### 项目定位
- **技术方向：** 多Agent系统架构、自然语言交互、工具集成
- **创新点：** 开箱即用、零代码配置、文化主题化设计
- **开源贡献：** 完整教程 + 一键部署脚本 + 真实案例分享

## 二、核心创新

### 1. 文化驱动的系统设计
借鉴明朝六部制度，将AI团队划分为14个专业"部门"：
- **兵部** — 软件工程、系统架构（Claude Opus）
- **户部** — 财务分析、成本管控（Claude Opus）
- **礼部** — 品牌营销、文案创作（Claude Sonnet）
- **工部** — DevOps、服务器运维
- **吏部** — 项目管理、团队协作
- **刑部** — 法务合规、安全审计

每个"部门"是一个独立的AI Agent，拥有专属的：
- 专业领域知识
- 独立的上下文记忆
- 特定的工具权限

### 2. 零代码、开箱即用
**传统多Agent框架的问题：**
- AutoGPT / CrewAI / MetaGPT 都是开发框架
- 需要编写大量代码来编排Agent
- 技术门槛高，不适合教学

**我的解决方案：**
- 一键部署脚本，5分钟完成安装
- 配置文件填写API Key即可使用
- Discord作为交互界面，@提及触发回复
- 适合教学演示和快速原型验证

```bash
# 一行命令完成部署
bash <(curl -fsSL https://raw.githubusercontent.com/wanikua/boluobobo-ai-court-tutorial/main/install.sh)
```

### 3. 智能协作机制
- **@mention触发：** @兵部 → 兵部响应，@everyone → 全员响应
- **自动Thread管理：** 复杂任务自动创建Thread，保持频道整洁
- **上下文继承：** Agent能读取项目记忆文件，持续积累知识
- **工具集成：** 不只是聊天，能写代码、管GitHub、写Notion、定时任务

### 4. 成本优化方案
- **免费服务器：** Oracle Cloud Always Free套餐（4核24GB ARM）
- **模型分层：** 日常对话用Sonnet，复杂任务用Opus，节省80%成本
- **真实可持续：** 我自己用了3个月，月成本¥100-300

## 三、技术实现

### 架构设计
```
用户（Discord）
    ↓
多个独立Bot（每个部门一个）
    ↓
Clawdbot Gateway（路由+工具调用）
    ↓
AI模型层（Anthropic/OpenAI/Qwen）
    ↓
工具层（GitHub/Notion/Browser/Cron）
```

### 关键技术点

**1. 多Agent隔离**
- 每个Agent有独立的session key
- 独立的上下文记忆存储
- 独立的工具权限配置

**2. 自动Prompt优化**
- 不需要手动调整Prompt
- 系统自动注入身份文件（SOUL.md / IDENTITY.md）
- 结合工作区文件生成完整系统提示
- Agent越用越懂项目

**3. 工具生态**
```javascript
// Agent自动调用工具
@兵部 帮我查一下GitHub上最新的issue
→ Agent调用GitHub CLI工具
→ 返回结构化结果

@户部 这个月服务器花了多少钱
→ Agent查询账单文件
→ 生成费用报表
```

**4. 自动化工作流**
```yaml
# Cron配置示例
jobs:
  - schedule: "0 22 * * *"    # 每晚22点
    task: "生成今日日报"
    agent: "main"
    
  - schedule: "0 20 * * 0"    # 每周日20点
    task: "生成本周周报"
    agent: "main"
```

## 五、技术挑战与解决

### 挑战1：Agent身份定义
**问题：** 如何让每个Agent真正"专业"？

**解决：**
- 设计专属的身份文件（identity主题）
- 自动注入领域知识（通过SOUL.md）
- 工作区文件持续训练（memory/目录）

### 挑战2：成本控制
**问题：** 多个Agent同时运行，API成本爆炸？

**解决：**
- 模型分层：Sonnet（日常）vs Opus（复杂任务）
- @mention机制：只有被提及才响应
- 自动Thread：避免重复处理同一问题

### 挑战3：上下文管理
**问题：** 如何让Agent记住项目历史？

**解决：**
- 分层记忆系统：
  - 短期：memory/YYYY-MM-DD.md（日志）
  - 中期：memory/weekly/（周报）
  - 长期：MEMORY.md（核心知识）
- Agent自动读取相关记忆
- 定期压缩归档
